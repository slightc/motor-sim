# 交互式仿真界面 · 状态录制器

一个**纯标准库**（无 numpy / flask / 任何第三方依赖）的电机仿真交互台：
在浏览器里选择控制算法、启停电机、加负载，并像浏览器性能调试工具一样
逐通道观察电机的**完整状态**。状态以满积分率（50 kHz）写入一个固定容量的
**环形缓冲**，始终保留**最近 10 秒**，超出即覆盖最旧。

## 运行

```bash
python3 interactive/server.py            # 默认 http://127.0.0.1:8090
python3 interactive/server.py 8090 0.0.0.0   # 指定端口/监听地址
```

> 注意：本环境里端口 8000 被代理占用，绑定会被信号杀死，故默认用 8090。
> 浏览器打开提示的地址即可。

## 功能对照需求

| 需求 | 实现 |
|---|---|
| 选择控制算法 | 顶栏下拉：有感 FOC（弱磁）/ 无感 FOC（反电动势）/ 开环 V/Hz。可选逆变器：理想 / SVPWM(10kHz)。改完点「应用 / 复位」 |
| 控制电机启停 | 「▶ 启动电机 / ■ 停止」。停止＝撤除驱动，电机在摩擦/负载下自由滑行（可观察惯性减速）；另有「⏸ 暂停时钟」冻结整个仿真 |
| 负载 | 「负载转矩」滑块（N·m），实时施加 |
| 转速给定 | 「转速给定」滑块（rad/s） |
| 录制最长 10s 完整状态、超出保留最后 10s | 满 50 kHz 写入 `array('d')` 环形缓冲，容量 = 10s；写满后覆盖最旧，永远是最近 10 秒 |
| 像性能工具一样观察各数据 | Canvas 多泳道时间线：滚轮缩放、拖拽平移、「跟随最新」实时滚动、悬停十字光标读各通道瞬时值。左侧按分组勾选 18 个通道 |

## 录制的「完整状态」通道（18 路）

相电流 `i_a/i_b/i_c`、`i_d/i_q`、转速 `ω_m实际/ω_m给定/ω_e`、电角 `θ_e`、
电磁转矩/负载转矩、铜损/铁损、反电动势/电压幅值、绕组/磁体/壳体温度。

## 内存账（回答「完全记录每秒多少内存」）

- 单样本：18 通道 + 时间戳 = 19 × 8 字节（C double）= **152 字节/样本**
- 满采样率 50 kHz（dt=20µs）：50000 × 152 ≈ **7.6 MB/s**
- 10 秒滚动窗口固定占用：`500001 × 19 × 8` ≈ **76 MB**（启动即预分配，不增长）

界面状态栏实时显示当前内存占用与缓冲填充度。

## 渲染原理（为什么 76MB 的缓冲也能流畅画）

后端按「每像素列取 min/max」抽稀（浏览器性能工具的同款套路）：无论窗口多宽，
每列只回传 `(min,max)` 两个数，既不漏尖峰、又把带宽压到与画布宽度无关。
缩放到几毫秒时直接读满分辨率原始样本，缩放到全 10 秒时按列聚合。

## 性能说明

纯 Python 在 50 kHz 满记录下约 **0.3–0.5× 实时**（状态栏「实时 N×」显示实测值）。
「仿真倍率」滑块是*目标*倍率，实际受单核算力上限约束。想要真正 1× 实时，可在
`sim_engine.py` 调大 `DT` 或加入录制抽稀（降低写入频率）。

## 架构

```
interactive/
  sim_engine.py     后台仿真线程 + 控制器/逆变器注册表 + 环形缓冲 + min/max 抽稀查询
  server.py         http.server 后端：控制 API + 窗口数据查询
  static/index.html 单文件 Canvas 前端（无构建步骤）
```

信号链复用 core：`Controller → Inverter → MotorPlant → Sensors → Controller`。
